# Directory organization

Each directory (work, cmake and build) have a purpose:

### Root dir

#### Files

* **README.md** - this file
* **run.bash**  - script for building and executing everything
* **CMakeLists.txt** - cmake file with project organization
* **clean.bash** - script to clean all products of building and making this ok for distribution

### work

This is the main software to test the funcionality

#### Files

* **work/main.cpp**
* **work/CMakeLists.txt**


### cmake

Some minor scripts in cmake to help with the build (actually just code format for a better visualization of the code)

#### Files

* **cmake/gwf_module.cmake** - the "fix_format" function will format source code for the files stored in the "SRC" variable on the build process

### build

Just to store the build products and dependecies generated by cmake software
